# Gerrit Commit Skill

This skill assists with committing code changes to Gerrit following Amlogic's commit message format.

## Commit Message Format

### Basic Format (All Projects)

```
module: subject [n/m]

PD#XXXXXX
BUG:XXXXX (optional)

Problem:
Detail info

Solution:
Detail info

Verify:
Detail info

Change-Id: XXXX
Signed-off-by: XXXX
```

**Requirements:**
- Each line < 80 characters
- Subject line format: `module: description [n/m]` (use [1/1] for single patch)
- PD#JIRA is mandatory (e.g., `PD#SWPL-54082`, cannot use `PD#NONE`)
- BUG: field is optional (for Google issue numbers)
- Sections: Problem, Solution, Verify (fixed format, first character not space)
- Change-Id: auto-generated by commit-msg hook
- Signed-off-by: generated by `git commit -s`

### Media Projects Special Format

For media projects (widevine, media_hal, ffmpeg-aml, etc.):

```
media_module: CF1 description [1/1]
```

**CF/CB Classification:**
- **CF (CL Feature)**:
  - CF0: Critical feature development or optimization
  - CF1: Important feature development or optimization
  - CF2: Normal feature development or optimization
- **CB (CL Bug)**:
  - CB0: Critical bug
  - CB1: Important bug
  - CB2: Normal bug

Media projects list: see [references/media-projects.md](references/media-projects.md)

### GFX Projects Special Format

For GFX projects, add `Test:` section after `Verify:`:

```
module: subject [n/m]

PD#XXXXXX

Problem:
Detail info

Solution:
Detail info

Verify:
Detail info

Test:
Detail info

Change-Id: XXXX
Signed-off-by: XXXX
```

GFX projects list: see [references/gfx-projects.md](references/gfx-projects.md)

### DRM Projects Special Signing

Some DRM projects must be signed off by DRM Auto Build:
- vendor/amlogic/mediahal_sdk
- vendor/amlogic/prebuilt/libmediadrm
- vendor/playready

Use: `Signed-off-by: DRM Auto Build <tao.guo@amlogic.com>`

## Workflow

**需要询问用户 username 的时机：**
1. **需要安装 commit-msg hook 时**（Step 3 方式 B 使用 scp）
2. **gerrit remote 不存在需要配置时**（Step 8 配置 Gerrit Remote）

### 1. Check Repository Context

```bash
git status
git remote -v
git log --oneline -5          # Reference for commit style
git diff --stat HEAD~1        # See what files changed
git diff HEAD~1 HEAD          # See actual diff
```

### 2. Determine Project Type

Check if current project is:
- **Media project**: See media-projects list
- **GFX project**: See gfx-projects list
- **DRM project**: Check if in DRM autobuild list

### 3. Get JIRA Issue Details

If user provides JIRA issue, fetch details with jira MCP tool:
```bash
get_issue issue_key="TV-201551"
```

Otherwise, ask user for JIRA issue.

### 4. Write Commit Message

Based on actual diff content, write:
- **Subject**: module + action (imperative mood) + [n/m]
- **Problem**: What issue this change addresses (from JIRA or diff analysis)
- **Solution**: What the change does (describe actual code changes)
- **Verify**: How it was tested
- **Test**: (GFX only) Test details

### 5. Stage and Commit

```bash
git add <files>
git commit -m "<message>"
```

#### Change-Id及Signed-off-by 获取方法（标准流程）

**Step 1: 填写 commit message 并提交**

```bash
git add <files>
git commit
```

提交时会弹出编辑器（或根据 template），填写格式：

```
audio: ap229 no sound [1/1]

PD#RSP-1883

Problem:
dts config error.

Solution:
sync from ap222

Verify:
yocto-ap222
```

**Step 2: 生成 Change-Id（必要）**

```bash
git commit --amend --no-edit
```

检查是否已生成 Change-Id：

```bash
git log -1
```

**Step 3: 如果生成不了 Change-Id，安装 commit-msg hook**

方式 A - 使用 curl：

```bash
curl -Lo .git/hooks/commit-msg http://scgit.amlogic.com:8080/tools/hooks/commit-msg
chmod u+x .git/hooks/commit-msg
```

方式 B - 使用 scp（需要用户提供 username）：

> **询问用户**: 请输入你的 Gerrit 用户名（如 shengyang.lin）：

```bash
# 获取正确的 git 目录（submodule 路径不同）
gitdir=$(git rev-parse --git-dir)

# 下载 hook（使用用户提供的 username）
scp -p -P 29418 <username>@scgit.amlogic.com:hooks/commit-msg ${gitdir}/hooks/
```

> **注意**: submodule 项目中，`.git` 目录可能在 `.git/modules/xxx` 下，务必使用 `git rev-parse --git-dir` 获取正确路径。

安装 hook 后，再次执行：

```bash
git commit --amend --no-edit
```

**Step 4: 生成 Signed-off-by 签名（必要）**

```bash
git commit -s --amend --no-edit
```

最终提交信息应包含：

```
audio: ap229 no sound [1/1]

PD#RSP-1883

Problem:
dts config error.

Solution:
sync from ap222

Verify:
yocto-ap222

Change-Id: Ixxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
Signed-off-by: Your Name <your.email@amlogic.com>
```

#### 更新已有 CL（复用 Change-Id）

Gerrit 以 Change-Id 区分 CL，只要基于同一个 Change-Id 就可以继续修改：

```bash
# 1. 下载对应 branch 的最新代码
git fetch gerrit

# 2. Cherry-pick 原 CL（保留 Change-Id）
git cherry-pick <commit-hash>

# 3. 修改代码
git add <files>

# 4. Amend 提交（保留原 Change-Id，生成新的 Patch Set）
git commit --amend --no-edit

# 5. Push 更新
git push gerrit HEAD:refs/for/<branch>
```

#### 解决 Merge 冲突后更新 CL

```bash
# 1. Cherry-pick 时产生冲突
git cherry-pick <commit-hash>
# Auto-merging xxx.c
# CONFLICT (content): Merge conflict in xxx.c

# 2. 解决冲突后，添加文件
git add <conflicted-files>

# 3. Amend 提交（保留原 Change-Id）
git commit --amend --no-edit

# 4. Push 更新
git push gerrit HEAD:refs/for/<branch>
# CL 会显示冲突已解决，新增一个 Patch Set
```

#### 添加 Signed-off-by

```bash
# 方法 1: 提交时添加 -s 参数
git commit -s -m "..."

# 方法 2: 对已有提交添加 sign-off
git commit --amend -s --no-edit

# 方法 3: 修改 sign-off（如需要改为 DRM Auto Build）
git commit --amend
# 手动修改 Signed-off-by 行
```

#### 常见问题排查

```bash
# 检查 hook 是否有执行权限
ls -la .git/hooks/commit-msg

# 添加执行权限
chmod +x .git/hooks/commit-msg

# 检查 hook 内容是否正确
cat .git/hooks/commit-msg | head -20

# 如果 submodule，确认使用的 .git 目录
git rev-parse --git-dir
```

### 6. 配置 Gerrit Remote

Push 前确保已正确配置 Gerrit remote（必须是 `scgit.amlogic.com:29418`）：

```bash
# 检查当前 remote
git remote -v

# 应显示如下格式（username 替换为你的用户名）：
# gerrit  ssh://username@scgit.amlogic.com:29418/<project-path> (fetch)
# gerrit  ssh://username@scgit.amlogic.com:29418/<project-path> (push)
```

如果未配置 gerrit remote，需要添加（添加前询问用户 username）：

> **询问用户**: 未配置 gerrit remote，请输入你的 Gerrit 用户名（如 shengyang.lin）：

```bash
# 格式: ssh://<username>@scgit.amlogic.com:29418/<project-path>
# 请将 <project-path> 替换为实际项目路径
git remote add gerrit ssh://<username>@scgit.amlogic.com:29418/<project-path>
```

### 7. Push 前确认!!!

配置好 remote 后，Push 前必须让用户确认以下信息：

```bash
# 1. 显示 Commit Message
echo "=== Commit Message ==="
git log -1

# 2. 显示 Remote 信息
echo ""
echo "=== Remote ==="
git remote -v | grep gerrit

# 3. 显示当前 Branch 信息
echo ""
echo "=== Current Branch ==="
git branch -vv | grep '^\*'
```

**需要用户确认的内容：**
1. **Commit Message** 格式是否正确（module、PD#、Problem/Solution/Verify、Change-Id、Signed-off-by）
2. **Remote** 是否正确（必须是 `username@scgit.amlogic.com:29418` 格式）
3. **Branch** 是否正确（目标分支与当前修改匹配）

**确认无误后再执行 Push。**

### 8. Push to Gerrit

```bash
# push 到 gerrit
git push gerrit HEAD:refs/for/<branch-name>

# 示例
git push gerrit HEAD:refs/for/master
git push gerrit HEAD:refs/for/amlogic-morty-4.9-tm2
```

> **注意**: 必须使用 `gerrit` remote（指向 scgit.amlogic.com:29418），而不是 `origin` remote。

## Common Errors

### Change-Id 相关错误

**错误: "missing Change-Id in commit message footer"**
- 原因: commit-msg hook 未安装或未执行
- 解决: 
  ```bash
  # 下载并安装 hook
  scp -p -P 29418 <username>@scgit.amlogic.com:hooks/commit-msg .git/hooks/
  chmod +x .git/hooks/commit-msg
  
  # 然后 amend 生成 Change-Id
  git commit --amend --no-edit
  ```

**错误: "Invalid Change-Id" 或 "Change-Id already used"**
- 原因: Change-Id 格式错误或与其他 change 冲突
- 解决:
  ```bash
  # 删除错误的 Change-Id，让 hook 重新生成
  git commit --amend
  # 手动删除 Change-Id 行，保存退出后会自动生成新的
  ```

**错误: submodule 中无法生成 Change-Id**
- 原因: submodule 使用不同的 .git 目录
- 解决:
  ```bash
  # 找到正确的 hooks 目录
  git rev-parse --git-dir
  # 例如输出: .git/modules/vendor/amlogic/media_modules
  
  # 在该目录下安装 hook
  scp -p -P 29418 <username>@scgit.amlogic.com:hooks/commit-msg \
    $(git rev-parse --git-dir)/hooks/
  ```

**错误: "duplicate Change-Id"（提交到新 branch 时）**
- 原因: 复用了其他 branch 上已有 change 的 Change-Id
- 解决:
  ```bash
  # 如果是全新的 change，删除 Change-Id 让 hook 生成新的
  git commit --amend
  # 删除 Change-Id 行
  
  # 如果是更新已有的 change，确保 Change-Id 与原 change 一致
  ```

### 其他常见错误

- **No files changed**: Only message updated, re-push
- **Sign-off check failed**: Use correct Signed-off-by for DRM projects
- **Subject format wrong**: Check if media/gfx project needs special format
- **Permission denied**: 检查 Gerrit 项目权限，确认有推送权限
